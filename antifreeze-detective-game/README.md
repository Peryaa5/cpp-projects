# Запах антифриза

Учебный графический проект на C++ — детективная 2D-игра с интерактивными сценами, диалогами и мини-играми.

## Описание

Проект представляет собой небольшую сюжетную игру-расследование. Игрок проходит сюжетные сцены, взаимодействует с объектами, собирает улики и в конце делает вывод о виновнике.

Игра создана с использованием библиотеки `graphics.h` / WinBGIm.

## Возможности

- графическое меню;
- сюжетные сцены;
- управление персонажем с клавиатуры;
- обработка мыши;
- диалоги;
- мини-игры;
- поиск улик;
- финальный выбор подозреваемого.

## Мини-игры

В проекте реализовано несколько интерактивных мини-игр:

- **шашки:** игрок может ставить и убирать шашки на доске;
- **телефон:** мини-игра по набору номера полиции, похожая на «однорукого бандита»;
- **дверь:** игрок должен несколько раз попасть мышью по случайно появляющемуся квадрату, чтобы открыть дверь;
- **лабиринт:** игрок управляет кругом и проходит через препятствия;
- **поиск кристаллов антифриза:** игрок ищет спрятанные синие элементы на экране;
- **финальный выбор подозреваемого:** игрок выбирает, кого считает убийцей, на основе найденных улик.

## Технологии

- C++;
- Code::Blocks;
- TDM-GCC 32-bit;
- WinBGIm / graphics.h.

## Что используется в коде

В проекте используются базовые конструкции C++ и элементы игровой логики:

- **функции** — код разделён на отдельные функции для меню, сцен, комнат, персонажей, мини-игр и отрисовки объектов;
- **прототипы функций** — в начале программы объявлены функции, которые дальше используются в коде;
- **переменные** — используются для хранения координат персонажа, положения мыши, состояния игры, выбранных действий и найденных улик;
- **флаги состояния** — переменные вроде `f`, `f1`, `FLAG`, `FLAGG`, `k1`, `k2`, `k3` используются для отслеживания прохождения сцен, найденных предметов и выбора игрока;
- **условные операторы `if / else`** — применяются для проверки действий игрока, столкновений, выбора объектов, найденных улик и переходов между сценами;
- **оператор `switch`** — используется для обработки нажатий клавиш, например стрелок, Enter и Esc;
- **циклы `while`** — используются для основных игровых циклов, ожидания действий пользователя и работы мини-игр;
- **циклы `do while`** — применяются для ожидания конкретного нажатия клавиши;
- **циклы `for`** — используются для отрисовки повторяющихся элементов, например клеток, линий, объектов и элементов лабиринта;
- **одномерные массивы** — применяются для хранения координат многоугольников при рисовании комнат, мебели и объектов;
- **двумерный массив** — используется для построения лабиринта;
- **случайные числа `rand()`** — применяются в мини-играх, например для случайного появления элементов;
- **обработка клавиатуры** через `getch()` и `kbhit()`;
- **обработка мыши** через `ismouseclick()` и `getmouseclick()`;
- **работа с координатами** — перемещение персонажа и взаимодействие с объектами реализованы через проверку координат;
- **отрисовка графических объектов** — комнаты, персонажи, предметы и интерфейс создаются с помощью графических примитивов;
- **простая игровая логика** — переходы между сценами, диалоги, проверка условий и финальный результат зависят от действий игрока.

## Используемые графические функции

В проекте используются функции библиотеки WinBGIm:

- `initgraph` — запуск графического режима;
- `cleardevice` — очистка экрана;
- `line` — рисование линий;
- `bar` — рисование закрашенных прямоугольников;
- `circle` — рисование окружностей;
- `fillellipse` — рисование закрашенных эллипсов;
- `drawpoly` — рисование многоугольников;
- `floodfill` — заливка области цветом;
- `setcolor` — выбор цвета линии;
- `setfillstyle` — выбор стиля заливки;
- `settextstyle` — настройка текста;
- `outtextxy` — вывод текста в заданных координатах;
- `getpixel` — проверка цвета пикселя, используется в лабиринте;
- `delay` — задержка для анимаций и обработки действий.
